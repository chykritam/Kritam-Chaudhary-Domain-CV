What is Full Stack Development?

Understanding the Big Picture

To me, being a full stack developer means understanding how the entire application comes together. It's split into two main areas:

The Front-End

This is everything the user sees in their browser. We build it using HTML for structure, CSS for styling, and JavaScript to make it interactive.

The Back-End

This is the server and database where information is actually stored and processed securely, usually built with languages like PHP or Python.

HTML (HyperText Markup Language)

Semantic Structure

I learned that HTML isn't just about putting text on a screen; it's about giving that text meaning using semantic tags. For my evidence, I built a functional contact form.

Here are a few things I focused on:

  • Using the correct input types (like type="email" and type="number").
  • Adding native browser validation using the required attribute.
  • Linking labels properly so the form is accessible to screen readers.
